## 0. 用語（このページで使う言葉）

| 言葉 | 意味 |
|---|---|
| **バックテスト** | 過去のデータで「もしこの戦略で売買していたら」を試すこと |
| **前進検証**（walk-forward） | 過去3年で売買ルールを決めて、次の1年で試す。窓をずらして16回繰り返す。**未来を見ない** |
| **in-sample** | 全期間を見た後で、その期間に当てはめること。**答えを見てから採点するようなもの** |
| **OOS**（out-of-sample） | 学習に使っていない期間で試すこと。前進検証はこれ |
| **イベントスタディ** | 「米中関税発表」のような**事件の前後**で指標が動いたかを調べる分析。バックテストとは別物 |
| **pre上昇** | 事件が**起きる前**に指標が上がっていたか。上がっていれば「予兆を捉えた」と言える |
| **イベントの客観的定義** | どの日を「事件」と呼ぶかを機械的なルール（例: VIXが○○超え）で決めること。**人間が選ぶと恣意的**になる |
| **ホールドアウト** | データの一部を**最初から隠しておいて**、手法が完成してから初めて試すこと |
| **窓依存** | 何日分のデータで計算するかで結果が変わってしまうこと |
| **p値** | 「偶然でもこれくらい起きる確率」。小さいほど偶然ではない。0.05未満が慣習的な基準 |
| **Δσ**（デルタシグマ） | 事件前後で指標が標準偏差の何個分動いたか。大きいほど強く反応 |
| **Sharpe（シャープ比）** | リスク1単位あたりのリターン。高いほど効率的 |
| **MaxDD（最大下落）** | 一番高かった時点から、どれだけ落ちたか |

## 2. やったけど載せなかった ＝「検証したが脆かったので載せていない」で答える

**載せなかったのは悪いことではなく、選別した証拠。** ただし何を選別したかは把握しておく必要がある。

### 2-1. イベントスタディ系（事件前後の分析）→ 核から外した

| 何を | 結果 | なぜ載せないか |
|---|---|---|
| **事件前の指標上昇（pre上昇）** | 手動28件で p=0.0018 だが、**窓を40日にすると消える**（p=0.787） | **窓の取り方で結論が変わる**＝脆い |
| **VIX→L¹ の先行性（lead-lag）** | 61lag補正後も p=2.9e-17 と最強 | **6月に核を「独立性＋バックテスト」に変更**したため不採用 |
| **中国A株での再現** | pre15 で Δσ=+1.30 (p=0.027, n=3)。**post-pre窓では負** | **イベント3件では結論を出せない**（Webサイトには正直に併記済み） |
| **統合指標α（12指標を束ねる）** | 5年で Δσ=+2.58 → **20年で −0.44（符号反転）** | **長期で再現しない** |
| **地政学イベントでの反応** | e_div Δσ=+0.47, **p=0.378（非有意）** | 効かなかった |
| **flip_rate（符号反転率）** | 当初 p=0.0001 と主張 → 検証で **p=0.692** | **再現しなかったのでサイトから削除済み** |

### 2-2. バックテスト系 → 期間・条件によっては効かない

| 何を | 結果 |
|---|---|
| **5年での前進検証** | ❌ **ランダムに負ける**（p=0.704）。10年でも互角（p=0.536） |
| **リターンでの勝負** | ❌ **全設定でαマイナス**（対持ち続け勝率18.75%）。「下落だけ浅い」が一貫した結論 |
| **他市場での売買** | ❌ 英FTSE αSharpe **−0.44**、独DAX −0.14、中国 **−0.47**。米国指数・日経・Appleでは勝つ |
| **銘柄を減らす** | ❌ 30本以下に減らすと Sharpe 0.999→0.48〜0.70に劣化 |
| **株価指数を抜く** | ❌ **シグナル消失**（Δσ +2.75 → −0.30） |

### 2-3. 理論系

| 何を | 結果 |
|---|---|
| **圏論的な10個の予想** | ⚠️ **支持5 / 反証5**。時間定常性・窓の連続性・4項同値などが数値的に反証 |

## 4. 発表での原則

| 聞かれたもの | 答え方 |
|---|---|
| **1. 載せているもの** | **数字で答える**。[NUMBERS.md](NUMBERS.md) にコード・データへのリンクあり |
| **2. 載せなかったもの** | 「**検証しましたが、窓を変えると消えるなど脆かったので載せていません**」 |
| **3. やっていないもの** | 「**未実施です。今後の課題です**」と認める。強がらない |

**「やったけど載せなかった」は弱みではない。** 全部載せて全部主張するより、脆いものを自分で外している方が強い。

## 5. この研究の射程（先に自分から言う）

> **米国株価指数を中心とした市場では、構造の崩れ（e_div）を使うと下落を有意に回避できる**
> （20年の前進検証で最大下落 −16%、ランダム500本中の上位1%、VIXにも勝つ）。
>
> ただし:
> - ①**個別の暴落を予知しているわけではない**（先行/後追いが混在、総合はmixed）
> - ②**株価指数を抜くとシグナルが消える**
> - ③**非米国市場では効かない**
> - ④**持ち続けには勝てない**（リターン・Sharpeで劣る）
>
> だからこれは**普遍的な法則でも予知でもなく、米国市場における「構造の異変の検知力」の実証**。

**先に限定する方が、後から限定させられるより強い。**

## 6. 正直に認識しておくべき構造的な弱点

**否定的な結果の出方に、気になるパターンがある**:

- **期間を伸ばす（20年）** → バックテストは有意になる（p=0.01）が、**イベントスタディの反応は減衰・符号反転**する
- **期間を縮める（5年）** → イベントスタディの反応は強いが、**バックテストはランダムに負ける**（p=0.704）

つまり「効いた」と言える期間が、**指標の種類ごとに食い違っている**。加えて `gamma_divergence_index.json` は「効果の大半は**2025年4月の大型関税5件**由来」と示し、`robustness_cluster_perm.json` は「n=5個別では非有意、クラスタでのみ有意」と示している。

**イベントスタディ側の発見が、実質的に2025年4月の単一イベント群に支えられている可能性は否定できない**。——これも**イベントスタディをポスターに載せていない理由の1つ**。ポスターの主張（独立性＋20年の前進検証）はこの問題の外にある。
